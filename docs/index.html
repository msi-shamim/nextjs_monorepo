Production-ready monorepos
in one command.

Generate a full-stack Next.js + NestJS/Express monorepo with Turborepo, your choice of ORM, auth, state management, Docker, payments, email, and more — fully configured and ready to develop.

npx @msishamim/create-next-monorepo my-app

Why This Tool?

Every full-stack project starts with the same 3-5 days of boilerplate: configuring workspaces, wiring databases, setting up auth, writing Dockerfiles, configuring linters. This tool eliminates that entirely.

You answer a few questions (or use flags), and in 30 seconds you have a production-grade monorepo with everything pre-configured, type-safe, and ready for pnpm dev.

Who is this for? Any developer building a full-stack TypeScript application — whether it's a SaaS, internal tool, or side project. From solo developers to teams. No monorepo experience required.

Getting Started

Prerequisites

  • Node.js version 18 or higher
  • pnpm version 9+ (recommended) — or npm, yarn, bun
  • Git (optional, for automatic git init)

Step 1: Run the generator

The simplest way — run with all defaults (NestJS, Tailwind, Prisma, NextAuth, Zustand):

npx @msishamim/create-next-monorepo my-app

Step 2: Navigate into your project

cd my-app

Step 3: Start development

pnpm dev

Your frontend is at http://localhost:3000 and your API at http://localhost:3001/api.

That's it! With zero flags, you get a complete monorepo with NestJS, Tailwind CSS, Prisma + Postgres, NextAuth.js, Zustand, Vitest, and shared UI components. Everything below is optional customization.

Project Structure

Every generated project follows this structure. Packages are added or removed based on your options.

my-app/
├── apps/
│   ├── web/              # Next.js 15 frontend (App Router)
│   └── api/              # NestJS or Express backend
├── packages/
│   ├── ui/               # Shared React components (Button, Card, Input)
│   ├── lib/              # Shared types, utils, constants, Zod validators
│   ├── config/           # Shared ESLint, TypeScript, Tailwind configs
│   ├── database/         # Prisma or Drizzle ORM (if enabled)
│   ├── cache/            # Redis client + service (if enabled)
│   ├── email/            # Email client + templates (if enabled)
│   ├── storage/          # File upload service (if enabled)
│   └── payments/         # Payment provider integration (if enabled)
├── .claude/              # AI agent skills + settings
├── turbo.json            # Turborepo pipeline config
├── package.json          # Workspace root
└── pnpm-workspace.yaml   # pnpm workspaces

Features

Each feature below is configurable via CLI flags. Every section shows you exactly which flag to use and what it generates.

Every project includes a Next.js 15 frontend using the App Router. This is not configurable — it's always included.

│   ├── globals.css      # Global styles
│   ├── not-found.tsx    # 404 page
│   ├── error.tsx        # Error boundary (client component)
│   └── loading.tsx      # Loading spinner

02. Backend Choice

Choose between two battle-tested Node.js frameworks for your REST API.

FlagValuesDefault
--backend or -bnestjs, expressnestjs

NestJS (default)

npx @msishamim/create-next-monorepo my-app --backend nestjs

Generates a full NestJS application with modules, controllers, services, and decorators:

apps/api/src/
├── main.ts                                # Bootstrap with CORS, validation, filters
├── app.module.ts                          # Root module
├── app.controller.ts                      # Health + info endpoints
├── app.service.ts                         # Business logic service
└── common/
    ├── filters/http-exception.filter.ts   # Global exception handler
    ├── guards/auth.guard.ts               # Bearer token guard
    ├── interceptors/logging.interceptor.ts # Request logging
    └── pipes/zod-validation.pipe.ts       # Zod schema validation

Express

npx @msishamim/create-next-monorepo my-app --backend express

Generates a lightweight Express application with routes, services, and middleware:

apps/api/src/
├── main.ts                                  # Server bootstrap
├── app.ts                                   # Express app factory with middleware
├── routes/health.ts                         # Health + info route
├── services/app.service.ts                  # Service class
└── common/
    ├── filters/error-handler.ts             # Error handling middleware
    ├── guards/auth.guard.ts                 # Auth middleware function
    ├── interceptors/request-logger.ts       # Logging middleware
    └── pipes/validate.ts                    # Zod validation middleware

03. API Style

Choose how your backend exposes its API. This layers on top of your backend choice — all combinations work.

FlagValuesDefault
--api-stylerest, graphql, trpcrest

REST (default)

No extra flag needed. Your backend already generates REST endpoints.

npx @msishamim/create-next-monorepo my-app

GraphQL (Apollo Server)

npx @msishamim/create-next-monorepo my-app --api-style graphql

Adds Apollo Server with resolvers, models (NestJS code-first) or typeDefs (Express schema-first). Access the playground at /graphql.

tRPC (End-to-End Type Safety)

npx @msishamim/create-next-monorepo my-app --api-style trpc

Generates the full tRPC stack — server router with example procedures, NestJS/Express adapter, and the Next.js client with typed hooks, provider, and server-side caller.

# Server (apps/api/src/trpc/)
trpc.ts         # initTRPC instance
router.ts       # Root appRouter with procedures
context.ts      # Request context
adapter.ts      # NestJS or Express adapter

# Client (apps/web/)
lib/trpc/client.ts      # createTRPCReact hooks
lib/trpc/provider.tsx   # TrpcProvider wrapper
lib/trpc/server.ts      # Server-side caller for RSC
app/api/trpc/[trpc]/route.ts  # API route handler

05. Database ORM

Choose your database layer. Both ORMs generate a schema, client singleton, and seed script.

FlagValuesDefault
--ormprisma, drizzle, noneprisma
--dbpostgres, mysql, sqlite, mongodbpostgres

Prisma (default)

npx @msishamim/create-next-monorepo my-app --orm prisma --db postgres

Generates packages/database/ with Prisma schema, client, and seed:

# Run migrations
cd packages/database
npx prisma migrate dev --name init

# Open Prisma Studio
npx prisma studio

Drizzle

npx @msishamim/create-next-monorepo my-app --orm drizzle --db postgres

Generates TypeScript-native schema definitions with inferred types:

# Push schema to database
cd packages/database
npx drizzle-kit push

# Open Drizzle Studio
npx drizzle-kit studio

No database

npx @msishamim/create-next-monorepo my-app --orm none
Docker integration When a database is selected, a docker-compose.yml is generated with the correct DB service (Postgres, MySQL, or MongoDB) for local development.

06. Authentication

FlagValuesDefault
--authnext-auth, custom, nonenext-auth

NextAuth.js v5 (default)

npx @msishamim/create-next-monorepo my-app --auth next-auth

Generates NextAuth v5 config with provider placeholders, session strategy, middleware protection, and type declarations. If Prisma or Drizzle is enabled, the appropriate adapter (@auth/prisma-adapter or @auth/drizzle-adapter) is automatically wired.

Custom JWT

npx @msishamim/create-next-monorepo my-app --auth custom

Generates a custom JWT auth system with login API route, middleware for protected routes, cookie-based sessions, and auth types.

No authentication

npx @msishamim/create-next-monorepo my-app --auth none

09. Shared Packages

Every project gets three shared packages that both apps import. This is always included.

packages/ui — React Components

Pre-built Button, Card, and Input components with variant/size props. Plus useMediaQuery and useDebounce hooks.

import { Button, Card } from '@my-app/ui';

packages/lib — Utilities & Types

Shared TypeScript types (ApiResponse, Result<T>), utility functions (formatDate, slugify), constants (HttpStatus), and Zod validators (emailSchema, paginationSchema).

import { ok, err } from '@my-app/lib/types';
import { emailSchema } from '@my-app/lib/validators';

packages/config — Shared Configs

Base ESLint (flat config), TypeScript (base, Next.js, Node presets), and Tailwind configs. All apps extend these.

10. Auto Version Resolution

At generation time, the CLI fetches the latest compatible version of every dependency from the npm registry. If you're offline, it falls back to tested hardcoded versions.

This means your generated project always starts with up-to-date dependencies — no stale package versions.

11. Doctor Command

Validate your monorepo's structure at any time. The doctor checks 100-170 directories and files (depending on your config) and reports what's missing.

Check structure

npx @msishamim/create-next-monorepo doctor

Auto-fix missing items

npx @msishamim/create-next-monorepo doctor --fix

Restorable files (README, LICENSE, configs, utilities, hooks) are regenerated with full template content. Non-restorable files get empty placeholders.

Config auto-detection Doctor automatically detects your project's config by reading package.json dependencies, schema files, and file structures. No manual config needed.

12. Workflow Guides

Step-by-step development guides that adapt to your detected project configuration.

# Overview of all workflows
npx @msishamim/create-next-monorepo workflow

# A: Build a reusable UI component
npx @msishamim/create-next-monorepo workflow a

# B: Build a Next.js page with state + API
npx @msishamim/create-next-monorepo workflow b

# C: Build a backend API endpoint
npx @msishamim/create-next-monorepo workflow c

Each guide prints file paths, code snippets, and checklists specific to your stack. For example, Workflow C shows NestJS module/controller/service if you're using NestJS, or Express route/service if you're using Express.

15. Docker

Docker templates are fully adaptive — they adjust to every option you've selected.

FlagValuesDefault
--dockerfull, minimal, nonenone

Full mode

npx @msishamim/create-next-monorepo my-app --docker full

Multi-stage Dockerfiles, nginx reverse proxy, health checks:

  • Dockerfiles adapt to package manager, Prisma generate, correct build commands
  • docker-compose.prod.yml includes DB service (postgres/mysql/mongodb), Redis, nginx, all env vars
  • nginx.conf routes /graphql, /api/trpc, /api/docs with WebSocket support

Minimal mode

npx @msishamim/create-next-monorepo my-app --docker minimal

Simple single-stage Dockerfiles + basic docker-compose. No nginx. Still adaptive to DB, Redis, and Prisma.
